Konvertieren von Datenspeichern

Der Datenspeicher-Konvertierungsassistent unterstützt Sie beim Konvertieren des Solaris-DHCP-Datenspeichers von einem Format in ein anderes. Wählen Sie "Datenspeicher konvertieren" im Menü "Service", um den Assistent zu öffnen.

Der Datenspeicher-Konvertierungsassistent fordert Sie auf, die folgenden Informationen einzugeben.

Datenspeichertyp
 Wählen Sie den neuen Datenspeichertyp.

Textdateien empfehlen sich für eine geringe Anzahl von DHCP-Clients, also weniger als 10.000.

Binärdateien eignen sich für Standorte mit sehr vielen Clients, an welchen nicht mehrere DHCP-Server auf gemeinsame Daten zurückgreifen müssen.

Wenn Sie bereits mit NIS+ als Name-Serice arbeiten und die Umgebung mehrere DHCP-Server enthält, empfiehlt sich NIS+, so dass alle DHCP-Server in der NIS+-Domain die Daten gemeinsam nutzen können.

Hinweis: Wenn der Server nicht bereits als NIS+-Client konfiguriert ist, können Sie die NIS+-Option nicht wählen. Um NIS+ für den Datenspeicher zu verwenden, beenden Sie den Assistent, konfigurieren Sie den Server als NIS+-Client, und führen Sie den DHCP-Manager erneut aus.

Datenspeicher konfigurieren
 Die hier geforderten Angaben hängen von dem gewählten Datenspeichertyp ab. Haben Sie Textdateien oder Binärdateien gewählt, geben Sie nun den Pfad zu den Daten ein (/var/dhcp ist Standard). Wenn Sie NIS+ verwenden, geben Sie die Domain des NIS+-Servers ein (die Domain dieses Servers ist Standard).

Alte Tabellen speichern Sobald feststeht, dass die Konvertierung erfolgreich abgeschlossen wurde, werden standardmäßig die zuvor vom Server verwendeten DHCP-Tabellen gelöscht. Möchten Sie nicht, dass diese Tabellen gelöscht werden, wählen Sie "Alte Tabellen speichern". Beachten Sie bitte, dass die Tabellen im Fall eines Problems bei der Konvertierung nicht gelöscht werden, auch dann nicht, wenn Sie die Option zum Speichern der alten Tabellen nicht gewählt hatten.
