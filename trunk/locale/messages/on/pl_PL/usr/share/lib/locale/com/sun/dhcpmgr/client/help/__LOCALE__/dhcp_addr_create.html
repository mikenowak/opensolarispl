Erstellen einer Adresse

Im Dialogfeld "Adresse erstellen" können Sie einen neuen DHCP-Client-Eintrag im gewählten DHCP-Netzwerk erstellen.

Verwenden Sie den Adreßassistenten, um einen Block Adressen zum DHCP-Netzwerk hinzuzufügen. Der Adreßassistent kann über das Menü "Bearbeiten" aufgerufen werden.

Die im Dialogfeld "Adresse erstellen" verfügbaren Einstellungen werden im Folgenden beschrieben.

IP-Adresse Geben Sie die IP-Adresse an, um sie neuen DHCP-Clients verfügbar zu machen. Die neue Adresse muss in der aktuellen DHCP-Netzwerktabelle eindeutig sein.
Client-Name (optional) Geben Sie den Hostnamen des Clients an. Wenn Sie diese Option verwenden, versucht der DHCP-Manager, einen Eintrag in der Tabelle Hosts zu erstellen.
Eigner-Server Geben Sie die IP-Adresse des DHCP-Servers an, dem die neue IP-Adresse gehört. Dieser Server ist dafür zuständig, zu Anfang auf die Anforderung einer IP-Adresszuordnung des DHCP-Clients zu reagieren.
Konfigurationsmakro Ein optionales Makro für den Server, das zum Abrufen von Client-Konfigurationsparametern aus der Datenbank dhcptab verwendet werden soll. Klicken Sie auf den Pfeil, um eine Liste mit den Makros anzuzeigen, die auf dem Server verfügbar sind. Wenn das gewünschte Makro noch nicht auf dem Server vorhanden ist, können Sie es über das Dialogfeld Makro erstellen definieren.
Kommentar (optional) Wenn Sie möchten, können Sie einen Textkommentar eingeben.

Klicken Sie auf "Leasing", um die Einstellung für den Leasing-Status und das Verfallsdatum zu konfigurieren. Die folgenden Einstellungen stehen zur Verfügung:
Client-ID Eine eindeutige Kennung des DHCP- oder BOOTP-Clients, der diese Adresse verwendet. Die Client-ID wird vom Anbieter des DHCP-Clients festgelegt. Für DHCP-Clients, die nicht unter Solaris laufen, erhalten Sie in deren DHCP-Client-Dokumentation mehr Informationen.

Bei Solaris-DHCP-Clients wird der Client-ID-String durch Verkettung des Typs der Netzwerk-Hardware und der Hardware-Adresse des Clients gebildet. So würde z. B. ein Client mit dem Hardware-Typ 01 (10-MB-Ethernet) und der Hardware-Adresse 8:0:20:11:12:b7 die Client-ID 010800201112B7 besitzen.

Ein Eintrag mit dem Client-ID-Wert 00 gibt an, dass die Adresse frei verfügbar ist, um sie einem anfordernden Client dynamisch zuzuordnen.

Reserviert Die Adresse ist für diesen Client reserviert und kann vom Server nicht neu zugeordnet werden.
Leasing-Policy Dynamisch: Der DHCP-Server ist für die Zuordnung und die erneute Zuordnung dieses IP-Adressbereichs zuständig.

Geben Sie die Verfallszeit für den Leasing-Zeitraum ein. Sie können das Verfallsdatum für den Leasing-Zeitraum im Format mm/tt/jjjj angeben.

Permanent: Der Leasing-Zeitraum ist unbegrenzt.

Nur BOOTP-Clients zuweisen Die gewählte IP-Adresse ist für BOOTP-Clients reserviert.
Adresse ist nicht verwendbar Diese IP-Adresse kann nicht verwendet werden. Sie können diese Einstellung zusammen mit der Option "Reserviert" verwenden, um einen Client am Booten zu hindern.
